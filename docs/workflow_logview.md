# Workflow Execution Logs

The Workflow Execution Logs feature provides a comprehensive, step-wise view of the validation, execution, learning, and orchestration history for a specific workflow run. This tool is designed to help developers debug complex multi-step workflows by surfacing internal agent decisions, communication, and learning insights that are otherwise hidden in the workspace file system.

## Overview

Logs are retrieved dynamically from the workspace's `runs/{runId}/logs/` directory. The system maps these raw log files back to the human-readable step titles, descriptions, and logical IDs defined in the workflow's `planning/plan.json`.

## Supported Log Types

The log viewer automatically detects and renders the following types of information, ordered by the natural workflow progression:

### 1. Execution Logs (`execution-attempt-N.json`)
- **Source:** The output of the primary execution agent for a step.
- **Contents:** The final execution result summary.
- **Deep Dive:** Includes a "View Full Conversation" feature that lazy-loads the `conversation.json` file, allowing users to see every tool call and LLM turn in a dedicated overlay.
- **Status:** Marks the step as "IN PROGRESS" if present without validation.

### 2. Validation Logs (`validation.json`)
- **Source:** Generated after an execution attempt to verify output against success criteria.
- **Contents:** Reasoning, severity-rated feedback items (CRITICAL, HIGH, etc.), and the raw validation agent response.
- **Visuals:** Status indicators (Green for success, Red for failure). Drives the overall "COMPLETED" or "FAILED" status of the step.

### 3. Learning Logs (`learning-execution.json`)
- **Source:** Generated by the Learning Agent after a step completes (Success) or fails (Failure).
- **Contents:** 
    - **Extracted Learning:** The specific insights, patterns, or corrections identified by the agent.
    - **Type:** "Success" (capturing best practices) or "Failure" (analyzing errors).
    - **Detail Level:** "Exact" (strict) or "General" (broad).
- **Deep Dive:** Includes a "View Full Conversation" button to see the Learning Agent's analysis process (`learning-conversation.json`).

### 4. Orchestration Logs (`orchestration-execution.json`)
- **Source:** Generated by steps using the Orchestration Orchestrator (Sub-Agents).
- **Format:** JSONL (JSON Lines) tracking the lifecycle of the orchestration.
- **Contents:** 
    - **Routing:** Which sub-agent was selected and why.
    - **Instructions:** **(New)** detailed instructions sent to the sub-agent.
    - **Success Criteria:** **(New)** specific success criteria for the sub-agent.
    - **Evaluation:** The final success evaluation of the orchestrated task.

### 5. Conditional Logs (`conditional-evaluation.json`)
- **Source:** Generated by Conditional Steps.
- **Contents:** The conditional question, the result of the evaluation, and the branch taken.

### 6. Decision Logs (`decision-evaluation.json`)
- **Source:** Generated by Decision Steps.
- **Contents:** The condition evaluated, the LLM's reasoning for the decision, and the resulting branch (`true`/`false`) that was selected.

## Architecture

### Backend (Golang)
- **Path:** `agent_go/cmd/server/workflow.go`
- **Logic:** 
    - Recursively scans the workspace `logs/` folder.
    - Uses `populateStepMetadata` to recursively map folder names (e.g., `step-1`, `step-3-true-0`) to logical step IDs (`original_id`) and titles from `plan.json`.
    - Reads `learning-execution.json` to populate the new `learnings` array.
    - Proxies file reading via the Workspace API to handle large conversation logs on demand.

### Frontend (React + TypeScript)
- **Component:** `ExecutionLogsPopup.tsx`
- **Service:** `agentApi.getExecutionLogs`
- **Features:**
    - **Visual Hierarchy:** Reordered sections to match workflow flow: Execution -> Validation -> Learning.
    - **Step Icons:** Type-specific icons (Network, GitBranch, User, Bot, Terminal) for quick identification.
    - **Logical IDs:** Displays the logical step ID (e.g., `fetch-pr-data`) alongside the title for better context.
    - **Semantic Theming:** Uses `bg-background`, `text-foreground`, and `border-border` for Light/Dark mode support.
    - **Lazy Loading:** Conversation logs (Execution & Learning) are fetched only when requested.

## How to Access
1. Open the **Workflow Canvas**.
2. Click the **Execution Logs** button in the bottom toolbar.
3. Select the desired **Run Folder** (e.g., `iteration-5`) to see the history for that specific execution.
